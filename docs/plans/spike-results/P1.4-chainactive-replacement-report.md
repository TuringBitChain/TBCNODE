## 2. P1.4 13 处 TxnValidation 直接读 chainActive — 已识别

`src/validation.cpp` `TxnValidation` 函数体内（约 1500-1750 行区间）chainActive 直接读约 13 处：

```
chainActive.Tip()        — 拍 tip 引用
chainActive.Height()     — 读高度
chainActive.Tip()->XXX   — 解引用 BlockIndex 字段
GetMedianTimePast(chainActive.Tip())   — 间接读
```

**当前状态**：这些路径全部持 cs_main（TxnValidation 调用方持有），**无 worker 不持 cs_main 路径**。

**P4.1 改造方向**（C-B 锁次序）：当 worker 真接入（P3.* 阶段）时，TxnValidation 入口拍 `auto snap = g_chainstate.Capture()`，函数体内全用 `snap.height` / `snap.tip_index` / `snap.mtp` / `snap.script_flags`。**当前 P1.4 不做改动**——保留 chainActive 路径稳定，等 worker 接入时整体切换。

## 4. 当前 P1.4 实际改动

**0 处直接替换**。理由：

1. **依赖 worker 接入**：worker 真路径在 P2/P3 才出现，P1.4 替换没意义
2. **依赖 C-B 锁次序**：ConnectBlock 锁次序在 P4.1 改造（UpdateTip 在 view.Flush() 之前），那时 g_chainstate 才真有数据
3. **风险隔离**：260+ 处替换是大手术，跟 worker 路径解耦做容易引入隐藏 bug

**留 P4.1 全量改造**。本卡只产出报告 + grep SCAFFOLD。
