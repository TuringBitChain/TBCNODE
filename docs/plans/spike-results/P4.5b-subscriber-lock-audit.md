## 3. 锁顺序矩阵（按订阅点）

### 订阅点 #3 wallet (HIGH)

```
worker commit (持 mempool.smtx unique)
    ↓ (在 v2.6.1 P3.7 后由 g_signal_dispatcher 异步分发，跨帧栈)
g_signal_dispatcher worker thread （没持 mempool.smtx 也没持 cs_main）
    ↓ subscriber callback
CWallet::TransactionAddedToMempool (持 cs_wallet)
    ↓ 老代码可能调 mempool.smtx 或 pcoinsTip
反向锁路径：cs_wallet → mempool.smtx → batchWriteMtx
```

**修复方案**：
- ✓ 方案 A（已实施）：v2.6.1 P3.7 SignalDispatcher 异步分发，跟 worker 不同帧栈 → wallet 持 cs_wallet 跟 worker 持 mempool.smtx 跨线程不直接互锁
- ⚠️ 方案 B（实施 if A 不够）：wallet 内部 hook 改异步队列（worker thread 内部），完全脱离 dispatcher

### 订阅点 #1 peerLogic (HIGH)

```
worker commit / ConnectBlock → BlockConnected2
    ↓ signal_dispatcher
PeerLogicValidation::BlockConnected (持 cs_main)
    ↓ relay block 给 peer
持锁顺序：cs_main → 内部 cs_vNodes / cs_inventory
```

**修复方案**：peerLogic 已经持 cs_main 操作，跟 v2.6.1 锁层级（cs_main level 0 最低）兼容。无需改动。

### 订阅点 #6 txindex (MED)

```
ConnectBlock → BlockConnected2
    ↓ signal_dispatcher
TxIndex::SyncTransaction (持 cs_main → 内部 m_db.Write)
```

**修复方案**：m_db 是 LevelDB 写，不参与 cs_main lock_hierarchy；m_db 内部锁是叶子，符合层级。无需改动。

### 订阅点 #7 journaling_block_assembler (MED)

```
mempool 变化 → assembler.UpdateMempool
持锁：cs_assembler → 读 mempool.smtx (shared)
```

**修复方案**：cs_assembler 是叶子锁；读 mempool.smtx (level 1) shared 不互斥 worker 阶段 4 unique commit（worker commit 持 unique 时 assembler shared 阻塞，commit 完即解锁）。无需改动。

### 订阅点 #2 zmq (LOW)

zmq 内部 zmqpub_mtx 是叶子锁，仅 socket send，不调 cs_main / mempool。**无反向锁风险**。

### 订阅点 #4/#5/#9 (LOW)

短生命周期或测试或叶子 cv，**无反向锁风险**。
