# P2.6 GC 线程 + 错峰 try_lock + 饥饿保护

**工时**：1 周  **owner**：TBD  **前置**：P2.1  **阻塞**：P3.1（GC 5ms 窗口跟 doubleCheck 配合）

## 1. 目标

实现 `ChainDispatcher::GcLoop`：16-shard 错峰 try_lock 清 COMMITTED 5ms 后 + ABORTED 立即；每 3ms 一 shard，连续 10 次 try_lock 失败强制取（防饥饿）。完整代码 detailed-design §2.4。

| KPI | 目标 |
