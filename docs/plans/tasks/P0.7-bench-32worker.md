# P0.7 性能 benchmark：32 worker 并发 ≥ 16x 单线程

**工时**：2-3 周  **owner**：TBD  **前置**：P0.6  **阻塞**：M0

## 1. 目标

量化 32 worker 并发 GetCoinConcurrent / BatchWrite / 混合负载吞吐，跟单线程对比，证实 P0 改造收益达预期。

| src/bench/coins_concurrent_bench.cpp | 新增 |
| docs/plans/spike-results/P0-bench-report.md | 报告归档 |

## 4. 验收 KPI

| KPI | 目标 |
|-----|------|
| 32 worker 无 BatchWrite 窗口吞吐 | ≥ 单线程 16x |
| 32 worker 含 BatchWrite 窗口（10min 周期）吞吐 | ≥ 单线程 8x |
| BatchWrite p99 | ≤ 200ms（重确认 P0.0b.2 KPI） |
| 单线程吞吐不退化 | ≥ P0 前 95% |
| 跨平台（x86_64 + ARM64） | 全过 |

## 5. 测试用例

```bash
./build/src/bench/bench_bitcoin --filter="(GetCoinConcurrent|Mixed)_.*" \
    --benchmark_threads=1,8,16,32 \
    --benchmark_repetitions=10 \
    --benchmark_format=json > p0-bench.json
python tools/analyze-bench.py p0-bench.json
```

## 6. 风险登记

| R | 缓解 |
|---|-----|
| R3 并发吞吐不达预期 | KPI ≥ 16x 阈值；不达标触发 M0 失败 |

## 7. 回滚条件

吞吐 < 16x → P0 改造投入产出比不足，触发 M0 失败评审 → 整套 v2.6.1 放弃可能性，沉没 11 + (8-9 月)。
