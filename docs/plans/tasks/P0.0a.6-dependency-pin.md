```markdown
## 漂移监控

每周 CI 跑 `dependency-drift-monitor.yml`：
1. 拉最新 Ubuntu 22.04 镜像
2. 检查 boost / glibc / kernel 实际版本是否仍 == 锁定值
3. 不一致 → 告警 + 触发 P0.0a.6 回归矩阵

## 升级协议

任何依赖升级（24 月内不可避免：Ubuntu 22.04 → 24.04 / boost 安全补丁 / kernel CVE 修复）：
1. 提 RFC PR：升级理由 + 影响分析
2. 跑完整 P0.0a.1+P0.0a.5+P0.0b.1 spike（缩水版，3 周）
3. 通过 → 锁定矩阵 PR 更新；不通过 → 不升级，stick 到旧版本直到 EOL
```

### 3.2 漂移监控 CI

```yaml
# .github/workflows/dependency-drift-monitor.yml
on:
  schedule:
    - cron: '0 0 * * 0'   # 每周日 UTC 00:00
  workflow_dispatch:
jobs:
  drift-check:
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/checkout@v3
      - run: |
          # 检查实际版本 vs DEPENDENCY-PIN-MATRIX.md
          GLIBC=$(ldd --version | head -1 | grep -oE '[0-9]+\.[0-9]+')
          BOOST=$(dpkg -l | grep libboost-all-dev | awk '{print $3}')
          KERNEL=$(uname -r | cut -d. -f1-2)

          PIN_GLIBC=$(grep glibc DEPENDENCY-PIN-MATRIX.md | awk '{print $4}')
          PIN_BOOST=$(grep boost DEPENDENCY-PIN-MATRIX.md | awk '{print $4}')
          PIN_KERNEL=$(grep kernel DEPENDENCY-PIN-MATRIX.md | awk '{print $4}')

          if [ "$GLIBC" != "$PIN_GLIBC" ] || [ "$BOOST" != "$PIN_BOOST" ]; then
              echo "DEPENDENCY DRIFT DETECTED"
              # 发 Slack/email 告警
              exit 1
          fi
```
