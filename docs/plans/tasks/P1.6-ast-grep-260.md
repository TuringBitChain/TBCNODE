每个调用点产出 3 个分类：

### F (final-audit) AST 漏报补充

clang AST 工具对以下场景可能漏报，**必须额外手工 grep 兜底**：

1. **宏展开**：`grep -nE "FOO_MACRO|BAR_MACRO" src/` 检查所有可能扩展为 chainActive 的宏定义
2. **模板实例化**：模板内调用 chainActive 在 AST 里可能只看到模板定义点，不是每个实例化点
3. **函数指针 / lambda 间接调用**：`grep -nE "std::function.*chainActive|\\[.*\\].*chainActive" src/`
4. **`using` / typedef 重命名**：`grep -nE "using.*=.*chainActive|typedef.*chainActive" src/`

每条补充 grep 输出归档到 chainactive-replacement-report.md 第二节。
