## 3. 代码改造

### 3.1 lock_hierarchy.h（仅常量）

```cpp
// src/validation/lock_hierarchy.h（新文件）
#ifndef BITCOIN_VALIDATION_LOCK_HIERARCHY_H
#define BITCOIN_VALIDATION_LOCK_HIERARCHY_H

#include <climits>

// v2.6.1 全局锁层级，强制低 level 先取
//
// 设计原则：
// 1. cs_main 是唯一 boost::recursive_mutex（其他全是 std::mutex / std::shared_mutex）
// 2. cs_main 持有期间可向下取所有锁
// 3. worker 验证路径不持 cs_main，仅取 level >= 1 的锁
// 4. 信号 dispatcher 跨线程异步派发，跟 worker 不同帧栈

constexpr int LEVEL_CS_MAIN          = 0;
constexpr int LEVEL_MEMPOOL_SMTX     = 1;
constexpr int LEVEL_BATCHWRITE_MTX   = 2;
constexpr int LEVEL_META_MTX         = 3;
constexpr int LEVEL_INFLIGHT_SHARD   = 4;
constexpr int LEVEL_WORKER_QUEUE     = 5;

// H-G 非破坏性增量：未标注 level 的现有 mutex 默认值
// 现有 401 callsite LOCK 宏调用零修改，编译期兼容
constexpr int LEVEL_DEFAULT          = INT_MAX / 2;

#endif
```

### 3.2 lock-hierarchy.md（节选）

```markdown
# 全局锁层级

## 锁列表

| Level | mutex | 类型 | 持锁者 | 持锁时长（典型）|
|-------|-------|------|-------|-------------|
| 0 | cs_main | boost::recursive_mutex | 主线程 / ConnectBlock / RPC blockchain.cpp | 1ms - 15s |
| 1 | mempool.smtx | std::shared_mutex | worker / ConnectBlock / RPC mempool 读 | shared 100µs / unique 1-5ms |
| 2 | pcoinsTip.batchWriteMtx | std::shared_mutex | worker shared / ConnectBlock unique | shared 30ns / unique 5-15s |
| 3 | pcoinsTip.metaMtx | std::shared_mutex | hashBlock 读写 | < 1µs |
| 4 | inflight_shard_mtx[16] | std::shared_mutex | dispatcher / worker callback | shared 100ns / unique 200ns |
| 5 | worker.queue_mtx | std::mutex | worker self / dispatcher push | < 1µs |
| INT_MAX/2 | LEVEL_DEFAULT（401 现有未标注 mutex） | 任意 | 任意 | 任意 |

## 强制规则

1. 持有 level N 时只能取 level > N 的锁
2. cs_main 是唯一 recursive，可同线程多次取
3. worker 验证不持 cs_main（level 0）
4. 401 现有 LOCK 调用点未标 level 视为 LEVEL_DEFAULT，跟所有标了 level 的 mutex 都不矛盾（INT_MAX/2 比 5 大）
5. 同 level 之间禁止持锁（除非用 std::lock 一次性获取多个）

## 验证机制

- 编译期：clang TSA 注解 `EXCLUSIVE_LOCKS_REQUIRED(...)`
- DEBUG_LOCKORDER build：thread_local lock stack 运行时检查
- release build：noop（不影响生产性能）

- wallet `TransactionAddedToMempool` → cs_wallet → mempool ?
- ZMQ publishing thread
- index/txindex SyncTransaction
- (P4.5b 完成后此节填充)
```
