# P2.7 异常捕获 + worker watchdog（软停语义）

**工时**：1 周  **owner**：TBD  **前置**：P2.3  **阻塞**：P3.1 P3.2

## 1. 目标

worker 异常路径（H5）：worker thread 抛异常 → MarkAborted（详 detailed-design §3.2 try/catch）+ promise.set_value(error)。watchdog 60s 软停语义：last_progress_us 超 60s → 警告 / 优雅重启。

## 2. 影响文件

| 文件 | 改动 |
|------|------|
| src/validation/per_chain_worker.cpp | ProcessItem try/catch + watchdog 接入 |
| src/validation/chain_dispatcher.cpp | watchdog 主循环 |
| src/test/worker_exception_tests.cpp | 新增 |

## 3. 代码改造

完整代码 detailed-design §3.2 try/catch + commit_succeeded 标志。

## 4. 验收 KPI

| KPI | 目标 |
|-----|------|
| worker 抛异常 → MarkAborted 立即 | 单元覆盖 |
| promise 必有 set_value（不丢失客户端响应）| 单元 |
| 60s 进度无更新 → watchdog 警告 | 集成 |
| 异常路径 ASan 0 leak | 测试 |

## 5. 测试用例

```cpp
TEST(WorkerException, ThrowDuringVerifyMarksAborted) { /* */ }
TEST(WorkerException, PromiseAlwaysSet) { /* */ }

```

## 6. 风险登记

| R | 缓解 |
|---|-----|
| R3 worker crash 死锁 inflight | MarkAborted + watchdog |
| R7 promise 丢失客户端响应 | always set_value |

## 7. 回滚条件

任一异常路径未覆盖 → 补单元 + revert 临界改动。
