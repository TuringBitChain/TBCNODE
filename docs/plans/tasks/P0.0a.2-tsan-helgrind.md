## 3. 代码改造

### 3.1 编译矩阵

```bash
# TSan build
cmake -B build-tsan -S . -Denable_tsan=ON -DBUILD_BITCOIN_WALLET=OFF
cmake --build build-tsan -j$(nproc)

# ASan build
cmake -B build-asan -S . -Denable_asan=ON -DBUILD_BITCOIN_WALLET=OFF
cmake --build build-asan -j$(nproc)

# UBSan build
cmake -B build-ubsan -S . -Denable_ubsan=ON
cmake --build build-ubsan -j$(nproc)
```

### 3.2 24h 长跑脚本

```bash
# tools/sanitizer-24h-soak.sh
set -euo pipefail
BUILD=$1   # build-tsan | build-asan
DURATION_H=24

export TSAN_OPTIONS="halt_on_error=0:second_deadlock_stack=1:suppressions=tools/sanitizer-suppression.txt"
export ASAN_OPTIONS="halt_on_error=0:detect_leaks=1:strict_string_checks=1"

start=$(date +%s)
$BUILD/src/test/test_bitcoin --log_level=test_suite > sanitizer-soak-$BUILD.log 2>&1 &
PID=$!

# Functional tests in loop for 24h
end=$(($start + DURATION_H * 3600))
while [ $(date +%s) -lt $end ]; do
    test/functional/test_runner.py --jobs=4 >> sanitizer-soak-$BUILD.log 2>&1
done

kill $PID 2>/dev/null || true

# 报告生成
grep -c "WARNING: ThreadSanitizer" sanitizer-soak-$BUILD.log > tsan-race-count.txt
grep -c "ERROR:" sanitizer-soak-$BUILD.log > error-count.txt
```

### 3.3 helgrind

```bash
# tools/helgrind-soak.sh
valgrind --tool=helgrind \
  --history-level=full \
  --read-var-info=yes \
  --num-callers=30 \
  --suppressions=contrib/valgrind.supp \
  ./build/src/test/test_bitcoin \
  --run_test=* \
  > helgrind-soak.log 2>&1
```

## 4. 验收 KPI

| KPI | baseline 目标 | 测量 |
|-----|---------|------|
| TSan 24h race count | 记录基线 N（不要求 N=0，但所有 race 必须分类） | grep WARNING |
| 已知 race 标 suppression | 100% 已知项进 suppression list | suppression.txt review |
| ASan 24h leak | 0 个新 leak（已有 leak 标记 baseline） | grep ERROR |
| UBSan 24h | 0 个 UB | grep runtime error |
| helgrind 24h race | 记录基线 M | grep "Possible data race" |
| 编译时间 | TSan +30%, ASan +40% 内 | time cmake --build |

**关键**：这卡不是要把 race 干到 0，是**建 baseline**。后续每张涉并发卡跟 baseline 比，新增 race 才是 v2.6.1 引入的。
